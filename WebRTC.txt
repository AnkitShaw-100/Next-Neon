WebRTC ek technology hai jisse hum browser to browser direct video, audio aur data transfer kar sakte hain bina kisi plugin ke.
Ye mainly UDP protocol use karta hai kyunki video/audio me low latency chahiye hoti hai.
Text chatting ya reliable data ke liye TCP use hota hai.

Important cheezein WebRTC me:

1. ICE (Interactive Connectivity Establishment):
Devices ke possible network paths check karta hai aur best connection choose karta hai.

2. STUN server:
Machine ko uska public IP address aur port discover karne me help karta hai.

3. TURN server:
Jab direct connection possible nahi hota (firewall/NAT ke wajah se) tab TURN server relay ke through data bhejta hai.

4. Signaling server:
Connection start karne ke liye dono peers ke beech me SDP (Session Description Protocol) aur ICE candidates exchange karta hai. Ye kaam hum mostly ek WebSocket/Node server se karte hain.

Connection setup process:

1. Machines STUN/TURN servers ke through apna public IP nikalti hain.
2. Signaling server ke through SDP aur ICE exchange hota hai.
3. Direct P2P connection ban jaata hai, fir data UDP par share hota hai.

Architectures:

Peer-to-Peer (P2P):
Sirf 2 log connect ho sakte hain.
Drawback: 2 se zyada users nahi handle ho paate.

Mesh:
Har ek user har dusre user se connect hota hai.
Drawback: scalability problem, network aur CPU load bahut badhta hai.

SFU (Selective Forwarding Unit):
Ek central server hota hai jo sabki streams receive karke forward karta hai.
Ye scalable hai aur large group calls ke liye best hai (Zoom, Meet, etc.).
